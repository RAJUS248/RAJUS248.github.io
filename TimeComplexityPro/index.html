Time complexity uses Big O Notation (O) to describe how the runtime of an algorithm grows as the size of the input (usually denoted as N) increases. It measures the rate of growth of operations.

Key Concepts

Big O Name Code Example
O(1)Constant TimeFixed operations (e.g., array access by index, dictionary lookup).
O(\log N)Logarithmic TimeSearch space is halved in each step (e.g., Binary Search).
O(N)Linear TimeSingle loop iterating N times (e.g., summing all elements in a list).
O(N \log N)Linearithmic TimeEfficient sorting algorithms (e.g., Merge Sort, Heap Sort).
O(N^2)Quadratic TimeTwo nested loops, both running up to N (e.g., Bubble Sort, comparing all pairs).
O(2^N)Exponential TimeBrute-force searching all subsets (e.g., solving the Tower of Hanoi recursively).
O(N!)Factorial TimeGenerating all possible permutations (e.g., brute-force Traveling Salesman Problem).

How to Analyze Code Complexity (Big O)

  1. Constant Time Operations: O(1) ‚Äî Occurs when the number of operations is fixed, regardless of the input size $N$ (e.g., array/list access by index, basic arithmetic).
  2. Single Loop: O(N) ‚Äî The code iterates through the input once. The complexity grows **linearly** with $N$.
  3. Logarithmic Growth: O(\log N) ‚Äî The loop counter is repeatedly **multiplied or divided** by a constant, effectively halving the problem space in each step (e.g., Binary Search).
  4. Nested Loops: O(N^2), O(N^3), etc. ‚Äî Two nested loops usually result in **Quadratic Time** $O(N^2)$. Three nested loops result in **Cubic Time** $O(N^3)$.
  5. Divide and Conquer (Merge/Quick Sort): O(N \log N) ‚Äî Occurs when an $O(N)$ operation is performed on a divided problem, often seen in efficient sorting algorithms.
  6. Exponential Growth: O(2^N) ‚Äî Typically occurs in brute-force recursive solutions where the problem space **doubles** with each addition to $N$ (e.g., recursive Fibonacci, finding subsets).
  7. Factorial Growth: O(N!) ‚Äî Extremely slow growth, involving the generation of **all permutations** of an input (e.g., brute-force Traveling Salesman Problem).
  8. Dominant Terms: When combining terms, keep the highest order term and drop constants (e.g., $O(N^2 + N) \rightarrow O(N^2)$).

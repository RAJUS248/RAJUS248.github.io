Time complexity uses **Big O Notation** ($O$) to describe how the **runtime** of an algorithm grows as the size of the input (usually denoted as **$N$**) increases. It measures the **rate of growth** of operations.

Key Concepts

Big O Name Code Example
$O(1)$Constant TimeFixed operations (e.g., array access by index).
$O(\log N)$Logarithmic TimeLoop counter multiplied/divided (e.g., Binary Search).
$O(N)$Linear TimeSingle loop iterating $N$ times.
$O(N \log N)$Linearithmic Time$O(N)$ loop containing an $O(\log N)$ operation (e.g., Merge Sort).
$O(N^2)$Quadratic TimeTwo nested loops, both running up to $N$.

How to Analyze Code

  1. **Single Loop:** $O(N)$
  2. **Nested Loops:** $O(N^2)$, $O(N^3)$, etc.
  3. **Logarithmic Growth:** If the loop counter is multiplied or divided by a constant, it's $O(\log N)$.
  4. **Dominant Terms:** Keep the highest order term and drop constants (e.g., $O(N^2 + N) \rightarrow O(N^2)$).
